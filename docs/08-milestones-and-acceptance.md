# 08-里程碑任务与验收

## 里程碑 1：工程与模型

周期建议：2-3 天。

任务：

- 初始化 monorepo。
- 初始化 Vue 3 前端。
- 初始化 Node.js 后端。
- 建立共享类型包。
- 定义核心 schema。
- 准备示例节点类型和拓扑 JSON。

验收：

- `pnpm dev` 可启动前后端。
- 前端能渲染空页面。
- 后端健康检查通过。
- 示例拓扑 JSON 校验通过。

## 里程碑 2：后端基础配置

周期建议：4-6 天。

任务：

- 数据库迁移。
- 节点类型 CRUD。
- 数据源 CRUD 和 preview。
- 拓扑 CRUD。
- 拓扑版本表。
- mock 运行态接口。

验收：

- 可通过接口新增断路器、实验室、文本节点。
- 可保存和读取拓扑。
- 可查询 `device_1001` 的电压、电流、开合状态。
- 关键接口有测试。

## 里程碑 3：编辑器 MVP

周期建议：6-10 天。

任务：

- 页面布局。
- Palette。
- GoJS Diagram。
- 节点、分组、文本、连线模板。
- 连线校验。
- 属性面板。
- 保存恢复。

验收：

- Palette 自动展示节点类型。
- 可以拖拽断路器到实验室。
- 可以从断路器端口拉线。
- 不符合端口规则的连线不能创建。
- 保存后刷新可恢复拓扑。

## 里程碑 4：数据绑定与运行规则

周期建议：5-8 天。

任务：

- RuntimeDataPool。
- BindingEngine。
- RuleEngine。
- 数据绑定编辑器。
- 规则编辑器基础版。
- 运行态页面。

验收：

- 文本节点 `电压：${U} V` 能显示运行数据。
- 断路器闭合时连线变绿色并启用流线。
- 电压小于 0 时连线变红并进入故障态。
- 实验室告警数为 0 时运行态隐藏。

## 里程碑 5：外部控制与事件

周期建议：3-5 天。

任务：

- `TopologyCanvasApi`。
- `TopologyEventEmitter`。
- 外部事件模型。
- 聚焦节点、聚焦分组。
- 设置节点和连线 runtime。

验收：

- 外部可调用 `setGroupVisible`。
- 外部可调用 `focusNode`。
- 点击节点时业务页面收到 `NODE_CLICK`。
- 点击连线时业务页面收到 `LINK_CLICK`。
- 不向业务暴露 GoJS 原始对象。

## 里程碑 6：校验发布与交付

周期建议：4-6 天。

任务：

- 保存前轻校验。
- 发布前强校验。
- 校验报告 UI。
- 发布流程。
- 操作日志。
- 权限点接入。
- 规则调试器第一版。

验收：

- 错误配置无法发布。
- 发布版本可在运行态读取。
- 校验报告能定位到节点、连线、规则。
- 规则调试器能展示命中原因。

## 第一版完整验收清单

- 可以在节点类型管理中新增“断路器”和“实验室”。
- 拓扑编辑器左侧 Palette 自动展示节点类型。
- 可以拖拽断路器到实验室分组中。
- 可以从断路器端口拉出连线。
- 不符合端口规则的连线不能创建。
- 可以给文字节点配置 `电压：${U} V`。
- 可以把 `U` 映射到 `device_1001.voltage`。
- 运行态聚合接口返回电压后，文字节点正确展示。
- 可以给连线配置“断路器闭合时运行”的规则。
- 运行态断路器状态变化后，连线颜色和流线变化。
- 可以给实验室配置“告警数为 0 时隐藏”的规则。
- 外部 API 可以控制实验室显示隐藏。
- 点击节点时，业务页面能收到 `NODE_CLICK` 事件。

## 主要风险

| 风险 | 处理建议 |
|---|---|
| 过早支持任意脚本 | 第一版只用结构化条件和模板变量 |
| 节点类型变更影响旧拓扑 | 节点类型版本化，删除字段前做引用检查 |
| 规则排查困难 | 第二阶段加入规则调试器 |
| GoJS 模板业务过重 | 业务规则放 RuleEngine，模板只绑定 runtime |
| 大图性能下降 | 限制动画数量、批量 commit、支持分组折叠 |

## 下一步建议

按顺序进入里程碑 1，先创建 monorepo 和共享类型包。共享模型稳定后，再同时推进后端配置接口和前端 GoJS 画布。
